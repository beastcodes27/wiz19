PesaLink Developer API

Integrate mobile money payments directly into your application using your personal API key. RESTful JSON API ΓÇö works with any language, framework, or platform.

REST Architecture
JSON Data Format
Bearer Auth Scheme
TZS Currency
Authentication

All API requests must include your personal API key in the Authorization header. Get your API key from your API Management dashboard after logging in.

Base URL
https://pesalink.online/api/
Authorization Header
Authorization: Bearer {YOUR_API_KEY}
≡ƒöæ Where to get your API key Log in to your account and go to API Management ΓåÆ click Generate New API Key. Your key starts with the site name prefix (e.g., Pesalink_xxxxxxxx...).
ΓÜá∩╕Å Keep your key secret Never expose your API key in client-side JavaScript or public repositories. If compromised, regenerate it immediately from your dashboard.
Quick Start

Get up and running with your first payment in under 5 minutes.

1

Get your API Key

Log in ΓåÆ navigate to API Management ΓåÆ click Generate New API Key. Copy the key securely.

2

Create a Transaction

Call POST /api/create-transaction with the recipient's phone number, amount, and their name. You'll receive a tranID instantly.

3

Track Payment Status

Use the tranID to call GET /api/status-transaction?tranid=... to check if payment is PENDING, COMPLETED, or CANCELLED.

4

(Optional) Set up a Webhook

Register a webhook URL in your dashboard to receive real-time payment status updates automatically ΓÇö no polling needed.

Complete Example

# Step 1: Create transaction
curl -X POST "https://pesalink.online/api/create-transaction" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"number":"0695123456","amount":5000,"name":"John Doe"}'

# Step 2: Check status (replace tranid value)
curl "https://pesalink.online/api/status-transaction?tranid=pay_aB3xYz9k" \
  -H "Authorization: Bearer YOUR_API_KEY"
// Step 1: Create transaction
$apiKey = 'YOUR_API_KEY';
$ch = curl_init('https://pesalink.online/api/create-transaction');
curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_POST           => true,
    CURLOPT_HTTPHEADER     => [
        'Authorization: Bearer ' . $apiKey,
        'Content-Type: application/json',
    ],
    CURLOPT_POSTFIELDS => json_encode([
        'number' => '0695123456',
        'amount' => 5000,
        'name'   => 'John Doe',
    ]),
]);
$response = json_decode(curl_exec($ch), true);
curl_close($ch);

$tranID = $response['data']['tranID'];

// Step 2: Check status
$ch2 = curl_init('https://pesalink.online/api/status-transaction?tranid=' . $tranID);
curl_setopt_array($ch2, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER     => ['Authorization: Bearer ' . $apiKey],
]);
$status = json_decode(curl_exec($ch2), true);
curl_close($ch2);

echo $status['data']['payment_status']; // PENDING / COMPLETED / CANCELLED
const API_KEY = 'YOUR_API_KEY';
const BASE    = 'https://pesalink.online/api';

// Step 1: Create transaction
const res = await fetch(`${BASE}/create-transaction`, {
  method:  'POST',
  headers: { 'Authorization': `Bearer ${API_KEY}`, 'Content-Type': 'application/json' },
  body:    JSON.stringify({ number: '0695123456', amount: 5000, name: 'John Doe' }),
});
const { data } = await res.json();
const tranID = data.tranID;

// Step 2: Poll for status
const poll = await fetch(`${BASE}/status-transaction?tranid=${tranID}`, {
  headers: { 'Authorization': `Bearer ${API_KEY}` },
});
const status = await poll.json();
console.log(status.data.payment_status); // PENDING / COMPLETED / CANCELLED

Transactions
POST /api/create-transaction Create Transaction

Initiate a mobile money payment to a recipient. The API responds immediately with PENDING status and a tranID. The actual payment processes asynchronously ΓÇö use the returned tranID to poll for the final status or listen on your webhook.

Phone numbers are automatically normalised from local format (0XXXXXXXXX) to international format (255XXXXXXXXX). The mobile network operator is auto-detected.

Request Body ΓÇö application/json
FieldTypeRequiredDescription
number string Required Recipient phone number. Accepts 0XXXXXXXXX or 255XXXXXXXXX. Min 10 digits.
amount number Required Amount in TZS. Must be a positive integer.
name string Required Recipient's full name. Used for transaction records.
Request Examples
curl -X POST "https://pesalink.online/api/create-transaction" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "number": "0695123456",
    "amount": 5000,
    "name":   "John Doe"
  }'

curl_setopt_array($ch, [

        'Authorization: Bearer YOUR_API_KEY',
        'Content-Type: application/json',
    ],
    CURLOPT_POSTFIELDS => json_encode([

        'amount' => 5000,
        'name'   => 'John Doe',

]);

GET /api/status-transaction Check Transaction Status

Check the current status of a transaction using its tranID. The tranID is returned when you create a transaction (format: pay_xxxxxxxx). You can only query transactions that belong to your account.

Query Parameters
ParameterTypeRequiredDescription
tranid string Required The unique transaction ID returned from create-transaction. Example: pay_aB3xYz9k
Request Examples
curl "https://pesalink.online/api/status-transaction?tranid=pay_aB3xYz9k" \
  -H "Authorization: Bearer YOUR_API_KEY"
$tranID = 'pay_aB3xYz9k';
$ch = curl_init('https://pesalink.online/api/status-transaction?tranid=' . urlencode($tranID));
curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER     => ['Authorization: Bearer YOUR_API_KEY'],
]);
$response = json_decode(curl_exec($ch), true);
curl_close($ch);
const res = await fetch(
  `https://pesalink.online/api/status-transaction?tranid=pay_aB3xYz9k`,
  { headers: { 'Authorization': 'Bearer YOUR_API_KEY' } }
);
const { data } = await res.json();
console.log(data.payment_status); // PENDING | COMPLETED | CANCELLED
Response 200
{
  "status":  "success",
  "message": "Payment status retrieved",
  "data": {
    "tranid":         "pay_aB3xYz9k",
    "payment_status": "COMPLETED",  // PENDING | COMPLETED | CANCELLED
    "amount":         5000,
    "network":        "AIRTEL",

GET /api/list-transaction List Transactions

Retrieve all transactions within a specified date range for the authenticated account. Results are scoped to your account only. Useful for reconciliation and reporting.

Query Parameters
ParameterTypeRequiredDescription
date_from string Required Start date in YYYY-MM-DD format (inclusive). Example: 2025-09-01
date_to string Required End date in YYYY-MM-DD format (inclusive). Example: 2025-09-30
Request Examples
curl "https://pesalink.online/api/list-transaction?date_from=2025-09-01&date_to=2025-09-30" \
  -H "Authorization: Bearer YOUR_API_KEY"
$ch = curl_init('https://pesalink.online/api/list-transaction?date_from=2025-09-01&date_to=2025-09-30');
curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER     => ['Authorization: Bearer YOUR_API_KEY'],
]);
$response = json_decode(curl_exec($ch), true);
const params = new URLSearchParams({ date_from: '2025-09-01', date_to: '2025-09-30' });

Webhooks

Webhooks let you receive real-time payment status notifications without polling. When a payment is completed or fails, we send an HTTP POST request to your registered URL.

Γä╣∩╕Å How to register a webhook Go to API Management in your dashboard ΓåÆ enter your public HTTPS URL under Webhook URL ΓåÆ save. Your endpoint must return HTTP 200 within 10 seconds or delivery will be retried.
Webhook Payload ΓÇö POST application/json
FieldTypeDescription
order_id string The tranID of the transaction (e.g. pay_aB3xYz9k)
payment_status string COMPLETED, FAILED, or CANCELLED
status string Alias for payment_status ΓÇö same value
Example Webhook Payload
{
  "order_id":       "pay_aB3xYz9k",
  "payment_status": "COMPLETED"
}
Webhook Handler ΓÇö PHP Example
// your-webhook-endpoint.php
$payload = json_decode(file_get_contents('php://input'), true);

$orderId = $payload['order_id']       ?? '';
$status  = $payload['payment_status'] ?? $payload['status'] ?? '';

if (empty($orderId) || empty($status)) {
    http_response_code(400);
    exit('Missing fields');
}

if ($status === 'COMPLETED') {
    // Credit the user's account or fulfill the order
    markOrderPaid($orderId);
} elseif ($status === 'FAILED' || $status === 'CANCELLED') {
    // Handle failure
    markOrderFailed($orderId);
}

http_response_code(200); // must return 200 or delivery retries
echo json_encode(['received' => true]);
Error Handling

All errors return a JSON body with status: "error" and a descriptive message. Use HTTP status codes to branch your error logic.

HTTP Code Meaning Common Cause
200 OK Request succeeded (GET endpoints)
201 Created Transaction was successfully created
401 Unauthorized Missing, invalid, or expired API key
404 Not Found tranid does not exist or doesn't belong to your account
422 Unprocessable Validation failed ΓÇö missing or invalid fields
500 Server Error Internal error ΓÇö contact support with your tranID
Error Response Structure
{
  "status":  "error",
  "message": "Missing required",
  "errors": {
    "number": ["Phone number is required."]
  }
}
